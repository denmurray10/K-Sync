# K-Sync Copy Style Guide

## Freshness

- Last reviewed: 2026-04-07
- Current truth: This guide reflects the current K-Beats tone across public pages, onboarding, live radio, and operator tools.
- Legacy context: Some older pages may still use flatter or more generic copy than the target tone described here.

## Purpose

Use this guide when writing headings, CTA text, labels, helper copy, status
messages, and onboarding text.

## Voice

K-Beats copy should feel:

- confident
- energetic
- premium
- fan-aware
- concise

It should not feel:

- overly corporate
- overly jokey
- generic SaaS
- vague or fluffy

## Headlines

- public headlines can be dramatic and editorial
- operator headlines should still be sharp, but clearer and more literal
- keep hero lines short and memorable

## Buttons And CTAs

- use strong verbs
- keep CTA labels short
- avoid vague labels like `Submit` when `Join`, `Start Listening`, or `Save Playlist` is clearer

## Labels And Metadata

- utility labels can be uppercase and compact
- metadata should support scanning, not compete with the main content
- use mono or utility-styled labels only where the design system already expects them

## Error And Success Messages

- be direct and specific
- public messages should stay calm and clear
- operator errors can be more explicit about what failed

## NotebookLM Rules

- If a page needs new copy, keep public copy expressive and operator copy precise.
- Prefer short, decisive CTA language over generic placeholder wording.
- If copy feels like a stock template, rewrite it to match K-Beats tone.
